# OfferHelper: 基于 Claude Skill 的岗位定制简历与德语求职信工作流

## 它是干什么的

`OfferHelper` 是一个面向单一候选人的 `Claude Skill`，用于根据具体岗位 JD 自动定制求职材料。

输入可以是：
- 岗位链接
- 粘贴的职位描述

输出可以是：
- 定制化简历 PDF
- 一页式德语 `Anschreiben`
- 或完整求职材料包

当前版本主要服务于单一候选人，并默认围绕固定简历模板和德语求职场景设计。

## 工作流

整个 skill 的目标不是“泛泛改简历”，而是把岗位要求映射到已维护的候选人资料，并在固定模板里进行有边界的定制化修改。

1. 分析 JD
   - 识别岗位名称、公司、核心要求、技术关键词和软技能要求。
   - 判断招聘方最看重的 3 个筛选标准。

2. 匹配候选人经历
   - 从候选人资料中提取可验证证据。
   - 决定哪些经历应该前置、哪些表达需要重写、哪些关键词值得进入技能区。

3. 改写简历模板
   - 仅修改约定的 4 个逻辑区域：
     - 副标题 / 目标岗位
     - 当前岗位要点
     - 过往经历补充块
     - 技能列表
   - 不改教育、语言、联系方式、照片和章节标题。

4. 导出简历
   - 在具备模板编辑与导出能力时生成最终 PDF。
   - 若导出能力不可用，则退化为结构化修改建议。

5. 生成德语求职信
   - 产出符合 DIN 5008 风格的一页式 `Anschreiben`。
   - 若无法直接生成 `.docx`，至少输出可直接落文档的最终正文。

## 前置条件

要让这个 skill 真正跑通，代理侧需要具备以下能力：

- 网页读取能力  
  用于抓取岗位链接中的职位描述。

- 模板编辑能力  
  用于修改既定简历模板中的目标区域。

- 文档生成能力  
  用于导出 PDF，以及生成或落地 `.docx` 格式求职信。

另外，这个公开仓库默认不包含以下内容：

- 真实手机号、邮箱、住址、LinkedIn
- Canva 私有模板链接
- 模板 design ID、page ID、element ID
- 私有候选人补充资料

这些内容应由你在私有配置或本地参考文件中维护。

## 如何自定义

如果你想把它改造成自己的版本，至少需要替换三类内容：

1. 候选人资料
   - 更新经历、教育、语言、优势和可验证事实。
   - 不要保留会诱发模型脑补的占位符。

2. 模板映射
   - 换成你自己的简历模板。
   - 重新维护“哪些区域可以改、哪些区域不能改”的逻辑映射。

3. 输出策略
   - 调整目标语言。
   - 调整简历与求职信格式。
   - 调整关键词优先级和压缩策略。

## 限制与风险

- 当前版本强依赖固定模板和单一候选人资料，不是开箱即用的通用简历产品。
- 候选人资料如果不完整，输出质量会明显下降。
- 如果私有模板配置缺失，skill 最多只能给出结构化改写方案，不能假装已经完成编辑。
- 如果某段经历缺少已验证事实，应该保守表达，而不是编造项目、指标或职责。
- 公开仓库必须脱敏；真实个人信息只应保留在私有版本中。

## 适用场景

这个仓库适合：
- 你已经有稳定的个人简历模板
- 你希望 Claude/代理根据不同岗位快速重写重点内容
- 你接受“强约束、高一致性”的求职材料工作流

它不适合：
- 多候选人共享使用
- 没有模板映射或没有私有配置的直接运行
- 需要完全通用、零配置的简历生成器
